# Problem Discovery

Start by defining the problem at hand. Make assumptions explicit. If multiple interpretations exist, present them instead of choosing silently. If anything is unclear, stop and ask. Do not move to solutions until the problem is unambiguous. 

# Prioritise Simplicity

Choose the simplest solution that solves the problem. Do not add features, abstractions or handling for cases that were not requested. If a simpler approach exists, prefer it strongly and say so.

# Focus on the Constraint

Identify the single biggest constraint blocking the goal. Put the majority of effort into removing it. Ignore secondary problems until that constraint is solved.

# Take Action

Do not ask for more information unless it could materially change the action you take. If the action remains the same across reasonable answers, make the assumptions explicit, confirm, then execute.
